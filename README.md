# TALI-H

TALI-H is the hand-arm module of the system. It is designed as the close-range manipulation model, meaning its primary function is to give the wearer additional working arms for gripping, holding, stabilising, tool use, machinery interaction, and quadrimanual coordination. From a design perspective, TALI-H should be treated as a pair of compact auxiliary robotic arms mounted from the thoracic docking points but engineered for precision and practical object handling rather than visual imitation of biological arms. The arms should remain short enough to work safely around the torso and natural arms, with fold-away resting positions that do not obstruct walking, sitting, or ordinary upper-body movement. From an engineering perspective, TALI-H requires modular wrists, interchangeable end-effectors, force sensing, compliant grasping, and a control system that allows the wearer to issue intent-level commands such as hold, release, grip, brace, stabilise, or pass object, while the onboard controller manages joint coordination, grip force, and collision avoidance. The hands do not need to be flesh-like; they should be selected according to task class, using adaptive robotic hands for dexterous manipulation and industrial grippers or clamps for heavy equipment support.

# TALI-W

TALI-W is the wing-arm module of the system. It is not a flying-wing model and should not be designed as an aerodynamic human-flight device in the first implementation. It is a deployable, foldable, load-bearing appendage system inspired by wing geometry, intended to provide wide-reach support, bracing, stabilisation, carrying assistance, and tool positioning. From a design perspective, TALI-W should have a broader and more structural visual language than TALI-H: spar-like segments, strong proximal roots, foldable distal sections, and compact stowage along the posterior body or flanks. From an engineering perspective, the wing-arms should prioritise structural stiffness, controlled folding, lockable joints, passive load support, and safe deployment envelopes. The end of each wing-arm may accept tools, pads, hooks, stabilisers, grippers, or specialised support attachments, but it should not be treated primarily as a humanoid hand. TALI-W is best understood as an external stabilising and reach-extension system that allows the wearer to create additional contact points with objects, work surfaces, machinery, or the environment.

# Shared TALI-Core Platform

The shared TALI-Core platform is the wearable base system that supports both TALI-H and TALI-W without merging them into one design. It is the structural, electrical, computational, and ergonomic foundation of the project. From a design perspective, TALI-Core must look and function like a serious thoracic-pelvic wearable chassis: a rigid posterior spine structure, pelvic load-transfer belt, thoracic stabilisation layer, breathable soft interface, adjustable sizing system, and protected routing for power, data, and control lines. From an engineering perspective, TALI-Core must transfer loads away from the ribs, shoulders, and neck and into the pelvis, posterior trunk, and lower body. It must contain the power bus, safety controller, sensor network, control computer, emergency stop circuit, harness pressure monitoring, module recognition, and candidate-specific calibration profiles. The purpose of TALI-Core is to allow different limb modules to be attached, removed, serviced, and upgraded while keeping the human interface, safety architecture, training software, and load path consistent.

# Universal Thoracic Dock Architecture

The Universal Thoracic Dock architecture is the mechanical and electronic interface that allows TALI-H and TALI-W modules to connect to the same TALI-Core platform. It should be treated as one of the most important engineering features of the entire project because it determines whether the system is modular, serviceable, upgradeable, and safe. From a design perspective, each dock should appear as a reinforced left or right thoracic root port positioned on the posterior upper-back frame, aligned with the intended auxiliary limb origin but mechanically tied into the pelvic load-transfer structure. From an engineering perspective, each dock must include a structural locking mechanism, keyed mechanical alignment, power connector, data connector, safety interlock, module identification chip, load sensor, optional pneumatic or vacuum channel, and emergency release. The dock should prevent incorrect attachment, prevent activation unless the module is fully locked, and automatically load the correct software profile for the attached module. TALI-H and TALI-W should therefore remain separate physical models, but both should become usable on the same wearable platform through the Universal Thoracic Dock.
